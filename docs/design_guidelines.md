# Design Guidelines

このドキュメントは、Canva などの参考テンプレートから抽出した「このアプリ用のスライド設計ルール」をまとめるためのものです。
現時点では参考画像が未配置のため、まずは観察テンプレートと初期ルールを先に定義しています。

目的:

- 参考画像をそのまま真似せず、再利用できるルールへ落とし込む
- prompt / preview / PPTX の判断基準を共通化する
- 社内資料向けの見やすさと実務性を優先する

## 1. 全体トーン

- 想定する資料タイプ:
  - 会議資料
  - 課題整理資料
  - 上司向け説明資料
  - 役員向け簡易提案資料
- 想定読者:
  - 部長
  - 本部長
  - 役員
  - プロジェクト責任者
- 全体の印象:
  - 端的
  - 落ち着いた配色
  - 結論先行
  - 余白多め
  - 装飾より可読性優先

## 2. 共通ルール

- タイトルの長さ:
  - 18〜28文字を目安にし、長くても 30 文字以内に抑える
- bullet 数:
  - 本文は 2〜3 点を基本にし、最大でも 4 点までに抑える
- 1枚の役割:
  - 1スライド1メッセージを維持する
- 情報量:
  - 読み上げなくても要点が伝わるが、説明文には寄せすぎない
- 表現トーン:
  - 抽象語より短い業務表現を優先する
  - 断定しすぎず、社内共有で使いやすい表現にする
- 強調方法:
  - 強調色は 1〜2 色に限定する
  - 強調はタイトル、ラベル、要点カードで行う
- 余白:
  - 情報を詰め込みすぎず、上下左右の余白を広めに取る
- レイアウト運用:
  - `layout2` は 1 本の論点を縦に読む本文用
  - `layout3` は 比較 / 課題と対応 / 原因と対策 の二分割用
  - `layout4` は 次アクションをカードで見せるまとめ用

なぜ分かりやすいか:

- 1枚ごとに「このページは何のためのページか」が最初に分かる
- タイトルが議題や章名のラベルとして機能し、読む前の迷いが少ない
- 本文は 2〜4 要素に分解され、1 ブロックあたりの意味が明確
- 見出し、小番号、ラベルが多く、視線の着地点が常にある
- 装飾より区切りが強く、どこから読めばよいか直感的に分かる
- まとめページで「次に何をするか」が独立して見える

## 3. 再現したい要素

- summary で結論と次アクションを強く見せる
- agenda は番号付きで流れが分かるようにする
- 比較スライドだけ 2 カラムにする
- type ごとに小さな role label を置き、読み手が一目で役割を理解できるようにする
- title は大きく、本文はカード単位で意味を分けて見せる

## 4. 再現しなくてよい要素

- 背景写真
- 装飾アイコン
- 派手なグラデーション
- 図形の多用
- 過度なアニメーション

### Reference 1

- ファイル:
  - `ホワイト　グレー　シンプル　出張報告書　プレゼンテーション.pptx`
- 向いている用途:
  - 報告資料
  - 出張報告
  - 会議向け進捗共有
- 良い点:
  - 目次のあとに目的、スケジュール、成果、課題、次アクションが素直に並ぶ
  - 1ページの役割が明確で、読み手が迷わない
  - 数字や成果は独立した要素として見せ、本文に埋め込みすぎていない
- 再現したい点:
  - `agenda -> content -> summary` の流れ
  - 成果、課題、改善点を別ページまたは別ブロックに分ける構造
  - 小さなページ番号や章番号で読書順を補助すること
- 再現しなくてよい点:
  - 実データ前提の細かな日付や訪問地の羅列
- 観察メモ:
  - タイトル位置:
    - 上部固定
  - 強調色:
    - グレー基調
  - bullet 密度:
    - 中程度
  - 余白の取り方:
    - 見出し帯を除き余白は広め
  - カード化されている要素:
    - 成果、課題、改善点
  - type / layout へ転用できるか:
    - `agenda`, `layout2`, `layout4`

### Reference 2

- ファイル:
  - `白　シンプル　年次報告書　プレゼンテーション.pptx`
- 向いている用途:
  - 年次報告
  - 役員向け説明
  - 実績共有
- 良い点:
  - 章タイトルページを挟むため、大きい資料でも迷いにくい
  - 指標スライドは大きな数字を見せ、補足説明は下に置いている
  - 長文があっても、最初に章名と主旨を示してから詳細へ入る
- 再現したい点:
  - 「章ラベル -> 要点 -> 補足」の順で読む構造
  - 重要指標は見出しと分けて大きく見せること
  - セクション単位で背景と要因を分割すること
- 再現しなくてよい点:
  - IR 資料寄りの長い説明文
  - 指標数の多さ
- 観察メモ:
  - タイトル位置:
    - 左上または中央で章番号とセット
  - 強調色:
    - 白地に濃色文字
  - bullet 密度:
    - 低〜高まで幅がある
  - 余白の取り方:
    - 章ページは広く、詳細ページはやや詰まる
  - カード化されている要素:
    - KPI、要因、サービス一覧
  - type / layout へ転用できるか:
    - `title`, `layout2`, 将来的な metric 表現

### Reference 3

- ファイル:
  - `白　ビジネス　プロジェクト進捗報告書　会議のプレゼンテーション.pptx`
- 向いている用途:
  - プロジェクト進捗報告
  - 週次、月次レビュー
- 良い点:
  - 進捗度、現状、次の予定が区切られており、会議で話しやすい
  - 数字、進行率、デザイン報告など、話題ごとにスライドが切れている
  - 角括弧ラベルや小見出しで、今どの観点を見ているかが分かる
- 再現したい点:
  - `layout3` を「現状 / 対応」や「進捗 / 課題」で使う判断
  - 本文に入る前のセクション導入
  - ラベルで読み筋を固定すること
- 再現しなくてよい点:
  - ビジュアル素材前提のページ
  - 英字ラベルの多用
- 観察メモ:
  - タイトル位置:
    - 左上固定が多い
  - 強調色:
    - 白地に濃紺、補助でグレー
  - bullet 密度:
    - 中程度
  - 余白の取り方:
    - カードとカードの間隔で整理
  - カード化されている要素:
    - 進捗度、デザイン報告、今後予定
  - type / layout へ転用できるか:
    - `layout2`, `layout3`, `summary`

### Reference 4

- ファイル:
  - `白　黒　シンプル　事業計画書　ビジネス　プレゼン　スライド.pptx`
- 向いている用途:
  - 事業計画
  - 提案資料
  - 投資判断向け簡易資料
- 良い点:
  - 競合分析や事業概要など、項目ごとに比較軸が明確
  - `Customer / Company / Competitor` のように比較見出しが先にある
  - テキストは短めで、表形式に寄りすぎず読める
- 再現したい点:
  - `layout3` の左右見出しを意味で付けること
  - 市場、競合、収支などテーマ別に粒度をそろえること
  - 事業概要ページで bullet を 3〜4 要素に制限すること
- 再現しなくてよい点:
  - ブランド名や個人名に依存する装飾
- 観察メモ:
  - タイトル位置:
    - 上部、やや中央寄り
  - 強調色:
    - 白黒中心
  - bullet 密度:
    - 中程度
  - 余白の取り方:
    - 余白はあるが情報ブロックは明確
  - カード化されている要素:
    - 比較軸、商品特徴、計画数値
  - type / layout へ転用できるか:
    - `layout3` の比較見出し

### Reference 5

- ファイル:
  - `青緑と白　プロフェッショナル　空　プレゼンテーション.pptx`
- 向いている用途:
  - 新規サービス説明
  - マーケティング提案
- 良い点:
  - アジェンダとセクション番号が強く、長めの資料でも今いる場所が分かる
  - 強調色は青緑系 1 色にほぼ限定され、視線誘導が安定している
  - 長文でも上段のタイトルが結論や観点を先に示している
- 再現したい点:
  - 強調色を限定し、タイトル・ラベル・線だけで使うこと
  - セクション番号を小さく固定表示すること
  - 一文説明が必要でも、最初の 1 行で主旨が分かるようにすること
- 再現しなくてよい点:
  - 背景写真や装飾要素の雰囲気依存
- 観察メモ:
  - タイトル位置:
    - 左上固定
  - 強調色:
    - 青緑 1 色
  - bullet 密度:
    - 低〜中
  - 余白の取り方:
    - 背景の余白を大きく使う
  - カード化されている要素:
    - セクション導入、論点見出し
  - type / layout へ転用できるか:
    - role label、accent 運用

## 6. 実装への反映先

### Prompt

- 反映したいルール:
  - 結論先行の短いタイトル
  - 本文 bullets は 2〜3 点を基本にする
  - `layout3` は二分割時だけ使う
  - summary は次アクションで終える
- 対象ファイル:
  - [`app/llm/prompts.py`](/home/sora/dev/llm-apps/slide_app/app/llm/prompts.py)

### Preview

- 反映したいルール:
  - role label を slide type ごとに表示する
  - summary を action card 風に見せる
  - `layout3` を 2 カラムパネルで表示する
- 対象ファイル:
  - [`app/templates/slides_fragment.html`](/home/sora/dev/llm-apps/slide_app/app/templates/slides_fragment.html)
  - [`app/static/style.css`](/home/sora/dev/llm-apps/slide_app/app/static/style.css)

### PPTX

- 反映したいルール:
  - preview と同じ role label を小チップで出す
  - `layout3` の見出しを二分割パターンに合わせる
  - summary を action card のまま出力する
- 対象ファイル:
  - [`app/services/pptx_service.py`](/home/sora/dev/llm-apps/slide_app/app/services/pptx_service.py)

## 7. 更新ルール

- 参考画像を追加したら、このファイルに観察メモを追記する
- その後、共通ルールへ昇格させるものだけを整理する
- 実装に反映したら、どのファイルへ反映したかを記録する
- Before / After 比較を残す時は、固定入力または `docs/experiments/` のログへリンクする
